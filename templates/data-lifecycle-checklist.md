# Data Lifecycle Checklist

## Before Sequencing

- [ ] Required sample metadata fields are defined.
- [ ] Sample identifiers are stable and unique.
- [ ] Data ownership/stewardship is clear.
- [ ] Consent, governance, or sharing constraints are recorded where relevant.

## During Data Generation

- [ ] Sequencing run metadata are captured.
- [ ] Run-level QC is recorded.
- [ ] Sample-level QC is recorded.
- [ ] Failed or excluded samples remain traceable.

## During Analysis

- [ ] Workflow version is recorded.
- [ ] Software/container versions are recorded.
- [ ] Reference/database versions are recorded.
- [ ] Parameters and thresholds are recorded.
- [ ] Inputs and outputs are traceable.

## After Analysis

- [ ] Final outputs are distinguished from intermediate files.
- [ ] Reports are stored in a controlled location.
- [ ] Sharing/export is recorded where required.
- [ ] Raw data retention rules are applied.
- [ ] Intermediate data cleanup rules are applied.

## Backup and Recovery

- [ ] Critical data are backed up.
- [ ] Backup copy is logically separated from primary storage.
- [ ] Restoration has been tested.
- [ ] Disaster recovery responsibilities are assigned.
