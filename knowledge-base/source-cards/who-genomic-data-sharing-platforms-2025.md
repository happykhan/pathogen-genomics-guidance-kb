## Extracted Notes

- Defines pathogen genomic data-sharing platforms as systems for submitting, storing, archiving, searching, accessing, and using pathogen genomic data and linked metadata.
- Sets out platform attributes covering transparency, governance, infrastructure/security, data scope, data submission, data curation, access, data provenance, interoperability, data use and benefit sharing, analytical/reporting capabilities, and sustainability.
- Supports a concrete metadata model at the platform level: minimum metadata should include sample information, high-level sampling strategy, sequencing strategy, bioinformatics methods, and attribution data, with optional pre-set metadata for richer epidemic or One Health analysis.
- Distinguishes manual and automated upload options, including API-mediated submission for larger datasets and options for limited internet connectivity.
- Treats infrastructure and security as platform attributes, including documented physical/cloud infrastructure, cybersecurity policy, resilience arrangements, archiving, scalability, and availability.
- Links richer metadata to more detailed inference while noting that richer metadata may require stricter access policies.

## Relevant Guide Sections

- Plan data sharing as a governed decision
- Integrate genomic, laboratory, and epidemiological data
- Treat identity and access as ongoing operations
- Separate active storage, backup, archive, and retention
- Design the sample-to-report data lifecycle

## Gaps

- This is a platform-level source. It does not prescribe which platform a country should choose for a particular pathogen or legal setting.
