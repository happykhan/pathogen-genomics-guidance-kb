# Access, Governance, and Security

## Core Principle

Pathogen genomics systems must manage who can see, upload, modify, analyse, export, and delete data. Access control and governance are ongoing operational responsibilities, not one-time setup tasks.

## Minimum Expectations

- Define user roles and permissions.
- Use institutional identity systems where feasible.
- Make onboarding, role changes, access review, and offboarding explicit.
- Log data access and analysis activity where risk requires it.
- Coordinate with central IT, cybersecurity, and information governance.
- Clarify incident response responsibilities before a breach or outage occurs.

## Key Risks

- orphaned accounts
- shared credentials
- unclear project ownership
- uncontrolled exports
- unmanaged collaborator access
- insufficient audit trails
- failure to revoke access when staff leave
