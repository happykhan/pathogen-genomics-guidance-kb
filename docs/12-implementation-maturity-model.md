## Purpose

Hold source-backed material for a future assessment or maturity model.

- PHA4GE vignettes and survey dimensions.
- Public repository `source-material/local/pha4ge-infrastructure-resources/docs/tiers_table.md`.
- `knowledge-base/extracted-notes/vignettes-and-survey-scoring.md`
- `knowledge-base/source-cards/phe-implementing-pathogen-genomics-case-study.md`
- Grant et al. 2018, `Implementing pathogen genomics: a case study`, Public Health England.

## Extracted/Adapted Text

The PHA4GE source supports comparative assessment across infrastructure dimensions such as future proofing, ease of use for administrators, ease of use for users, data provenance and management, access control, external access requirements, flexibility, and scalability.

The public repository contains a tier table that sketches capability levels, but it is currently treated as draft source material, not a validated assessment instrument.

The PHE case study gives an implementation timeline and service-transition example that could inform maturity staging, but it does not define a reusable maturity rubric.

## Implementation Notes

Do not present a maturity model as authoritative until the assessment dimensions, scoring method, and intended use are reviewed.
