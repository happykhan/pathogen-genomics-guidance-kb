# Data Lifecycle

## Core Principle

Pathogen genomics data should be managed as a lifecycle, not as a collection of files. Laboratories need defined processes for data creation, ingestion, QC, processing, storage, analysis, interpretation, sharing, archiving, retention, and deletion.

## Lifecycle Stages

1. Sample and metadata capture
2. Sequencing run setup
3. Primary data generation
4. Demultiplexing and initial QC
5. Data ingestion into analysis/storage systems
6. Bioinformatics processing
7. Result QC and interpretation
8. Reporting and data sharing
9. Archiving and retention
10. Review, deletion, or long-term preservation

## Minimum Expectations

- Every dataset should have a known owner or steward.
- Raw data, intermediate data, final results, and reports should be distinguishable.
- Metadata should be captured early and carried through analysis.
- Data movement between systems should be documented.
- Retention and deletion rules should be explicit before data volume becomes unmanageable.
- Data lifecycle choices should be aligned with confidentiality, integrity, availability, and public health use.
